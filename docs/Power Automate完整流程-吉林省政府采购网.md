# Power Automate 完整采集流程 - 吉林省政府采购网

## 问题说明

**XPath 报错原因**：HTML 不是严格的 XML 格式，Power Automate 的"执行 XPath 表达式"操作要求输入必须是有效的 XML 文档。

**解决方案**：使用 **"执行正则表达式"** 替代 XPath。

## 关键技巧

### 1. 处理多个同名字段

页面中有多个"名 称"、"联系方式"等字段，解决方法：

**方法一：先提取信息块，再从块中提取**
```
步骤1: 提取采购人信息块（用正则定位区块）
步骤2: 从采购人信息块中提取"名 称"
步骤3: 提取代理机构信息块
步骤4: 从代理机构信息块中提取"名 称"
```

**方法二：使用数组索引**
```
操作: 执行正则表达式
参数:
  - 正则表达式: 名\s*称[：:]([^<\n]+)
  - 查找所有匹配项: 是
  - 匹配结果: %所有名称%

操作: 设置变量
参数:
  - 采购人名称: %所有名称[0]%  (第一个)
  - 代理机构名称: %所有名称[1]%  (第二个)
```

### 2. 处理中文冒号

中文的冒号可能是全角 `：` 或半角 `:`，正则中用 `[：:]` 匹配两种情况。

### 3. 处理空格

HTML 中可能有多个空格，用 `\s*` 匹配任意数量的空白字符。

```

```

```
  循环内操作:
    - 拼接完整URL
    - 调用详情页采集流程
    - 保存数据
    - 返回列表页
```
